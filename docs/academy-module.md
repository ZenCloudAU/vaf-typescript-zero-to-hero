# Velocity Academy Module Proposal

## Module title

TypeScript for AI-Native Architecture Practitioners

## Placement

This module belongs under:

- Agentic AI for Architecture
- The Agentic Architect library track
- Practitioner Resources
- StudioSix Labs production pipeline

## Positioning

This is not a generic coding bootcamp. It teaches enough TypeScript for architecture practitioners to inspect, direct, and govern AI-generated digital tools.

## Target learner

- Enterprise architects
- Solution architects
- Business architects working with AI tools
- PMO and delivery leads building lightweight governance tools
- Vibe coders who need stronger technical control

## Learning outcomes

Learners will be able to:

1. Explain what TypeScript is and why it matters.
2. Read basic TypeScript syntax.
3. Understand function inputs and outputs.
4. Model architecture decisions using interfaces.
5. Debug simple type errors.
6. Use Cursor and VS Code safely.
7. Create a small decision-support utility.

## Course structure

### Week 1: Foundations

- What is code?
- What is JavaScript?
- What is TypeScript?
- Installing Node.js, VS Code, Cursor, and Git
- Running the first TypeScript file

### Week 2: Types and functions

- Strings, numbers, booleans
- Functions
- Return types
- Reading compiler errors

### Week 3: Objects and interfaces

- Objects as business records
- Interfaces as contracts
- Optional fields
- Literal types

### Week 4: Arrays and decisions

- Lists of data
- Filtering and mapping
- Simple scoring models
- Decision-first coding patterns

### Week 5: Mini project

Build a simple Velocity-style decision scoring tool.
